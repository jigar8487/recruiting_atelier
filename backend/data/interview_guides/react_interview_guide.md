# React Senior Interview Guide

## Goals
Validate that the candidate can:
1. Architect a non-trivial React app
2. Reason about performance (re-renders, memoization, bundle size)
3. Write maintainable TypeScript and tests

## Topics to cover
- Component composition and prop-drilling avoidance
- State management trade-offs (Context, Redux, Zustand, Jotai)
- React hooks: useMemo, useCallback, useEffect pitfalls
- Server components vs client components (Next.js App Router)
- Accessibility: keyboard nav, ARIA roles, focus management
- Testing: React Testing Library philosophy

## Signal levels
- **Strong**: candidate volunteers performance pitfalls and discusses fixes proactively
- **Good**: candidate answers correctly when asked, but doesn't volunteer depth
- **Weak**: candidate misses re-render or memoization basics
